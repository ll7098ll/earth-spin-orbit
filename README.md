# 🌌 지구의 운동 가상 실험실 (Earth Motion Virtual Lab)

초등학교 6학년 과학 **'4. 지구의 운동'** 단원의 교육과정을 완벽히 반영한 웹 기반 3D 및 2D 천체 가상 실험실 플랫폼입니다.  
학생들과 교사가 교과서 속 자전 및 공전 모형 실험을 웹 브라우저에서 관찰하고, 핵심 이론을 도식화 형태로 학습하며, 교과서 기반의 고도화된 퀴즈를 풀 수 있도록 설계된 단일 페이지 애플리케이션(SPA)입니다.

## 🛠️ 기술 아키텍처 및 구현 상세 (Technical Architecture)

본 프로그램은 외부 플러그인 설치 없이 크롬북을 비롯한 저사양 교육용 기기에서 구동되도록 경량화된 웹 표준 기술로 개발되었습니다.

* **프레임워크**: React 19.2.6 (컴포넌트 기반 상태 관리 및 SPA 라우팅)

* **애니메이션 루프**: `requestAnimationFrame`을 활용한 부드러운 60fps 렌더링 프레임 보장
* **인터페이스 스타일링**: Vanilla CSS (우주 테마의 네온 Glow 및 투명 글래스모피즘 디자인 적용)
* **컴포넌트 의존성**:
  - `lucide-react`: 직관적인 조작기용 모노톤 라인 아이콘 팩
  - `canvas-confetti`: 퀴즈 완점 달성 시 성취감을 극대화하는 꽃가루 파티클 효과

## 📐 수학적 좌표 및 동기화 원리 (Mathematical Synchronization)

시뮬레이션의 핵심은 **3D 우주 관점(지구의 실제 운동)**과 **2D 관측자 관점(천체의 겉보기 운동)** 간의 수학적 동기화입니다.

### 1. 관측자 핀(Pin)의 3D 표면 좌표 매핑
지구본 3D 모델(구형 geometry) 위에 대한민국 서울의 위치를 표시하기 위해 구면 좌표계(Spherical Coordinates)를 직각 좌표계(Cartesian Coordinates)로 변환하여 핀의 위치를 계산합니다.
지구의 반지름을 $R$, 위도를 $\phi$ (서울: 약 북위 $37.56^\circ$), 경도를 $\theta$ (서울: 약 동경 $126.97^\circ$)라고 할 때, 3D 공간 상의 좌표 $(x, y, z)$는 다음과 같이 결정됩니다.

$$x = R \cdot \cos(\phi) \cdot \sin(\theta)$$
$$y = R \cdot \sin(\phi)$$
$$z = R \cdot \cos(\phi) \cdot \cos(\theta)$$

텍스처 매핑과의 정밀한 시각적 동기화를 위해, 3D 상의 빨간색 핀 좌표는 지구본 이미지 텍스처의 한반도 픽셀 위치(가로 세로 비율 매핑)와 수학적으로 완벽히 일치하도록 보정되었습니다.

### 2. 3D 자전각과 2D 남중 시야의 위상차($\pi$ offset) 보정
Three.js의 `SphereGeometry`에서 자전각(Y축 회전각) $\theta_{earth}$가 변화할 때, 지구 표면의 관측자가 바라보는 남쪽(정면) 하늘에 특정 천체(태양 또는 별)가 도달하는 시점을 동기화해야 합니다.
컴퓨터 그래픽스의 텍스처 좌표계 $U$축 매핑과 Three.js의 XZ 평면 회전 기준선 사이에는 $180^\circ$ ($\pi$ 라디안)의 수학적 위상차(Phase Shift)가 존재합니다. 따라서 관측자가 남쪽을 정면으로 바라볼 때 천체가 정확히 화면 중앙(자오선)에 위치하도록 다음과 같은 회전 보정 공식을 적용합니다.

$$\theta_{observer} = \theta_{earth} - \pi$$

이 위상차 보정을 통해 지구본 위의 빨간색 핀이 태양 또는 별과 일직선으로 마주 보는 시점(정오 또는 자정)에, 우측의 2D 관측자 시야 캔버스에서 해당 천체가 남쪽(가운데) 정중앙에 정확히 남중하도록 동기화가 이루어집니다.

## 💻 크롬북(Chromebook) 최적화 디자인

학교 교실 현장에서 널리 사용되는 크롬북의 하드웨어 및 화면 제약을 극복하기 위해 다음과 같은 UX 디자인 패턴을 설계했습니다.

1. **Flex-Fit 단일 화면 레이아웃**:
   - 세로 해상도가 낮은 크롬북 특성(`max-height: 740px`)을 고려하여 브라우저 스크롤바가 발생하지 않는 화면 맞춤 디자인을 적용했습니다.
   - 좌측에 인터랙티브 캔버스 뷰포트를 배치하고, 우측에 3탭 사이드바를 배치하여 시선의 분산을 방지하고 조작과 관찰이 동시에 가능하도록 설계했습니다.
2. **WebGL 리소스 최적화 및 가비지 컬렉션**:
   - 각 실험실 컴포넌트가 언마운트(Unmount)될 때 Three.js의 `WebGLRenderer`, `Geometry`, `Material`, `Texture`가 메모리에서 명시적으로 해제(`dispose()`)되도록 라이프사이클을 관리합니다. 이를 통해 메모리가 제한된 크롬북에서 장시간 구동해도 브라우저가 느려지거나 튕기는 현상을 원천 방지합니다.
3. **고대비 가독성 테마**:
   - 어두운 교실 환경이나 밝은 창가에서도 텍스트를 명확히 판독할 수 있도록 우주 감성의 어두운 배경 위에 대비가 명확한 네온 컬러 계열의 하이라이트 텍스트(Glow 효과)를 적용했습니다.

## 🚀 시작하기 및 실행 방법 (Getting Started)

### 1. 개발 환경 설치
로컬 컴퓨터에 [Node.js (v18 이상 권장)](https://nodejs.org/)가 설치되어 있어야 합니다. 프로젝트 루트 폴더에서 다음 명령어를 실행하여 필요한 종속성 라이브러리를 설치합니다.
```bash
npm install
```

### 2. 로컬 개발 서버 구동
코드 수정 사항을 실시간(HMR)으로 확인하며 개발 및 테스트를 진행합니다.
```bash
npm run dev
```
구동 후 브라우저에서 `http://localhost:5173`으로 접속합니다.

### 3. 배포용 파일 빌드
웹 서버 및 Netlify 호스팅 서비스에 배포할 수 있는 고도로 압축/최적화된 정적 빌드 파일을 생성합니다.
```bash
npm run build
```
빌드가 완료되면 프로젝트 루트에 `dist/` 폴더가 생성되며, 이 폴더의 내부 파일들이 최종 배포용 에셋입니다.

## ☁️ 배포 안내 (Deployment)

본 프로젝트는 Netlify 플랫폼과의 연동 및 드래그 앤 드롭 업로드 배포를 정식 지원합니다.

* **배포 폴더(Publish directory)**: `dist`
* **단일 페이지 앱(SPA) 라우팅 지원**: 루트에 탑재된 `netlify.toml` 설정을 통해 새로고침 시 발생하는 404 오류가 자동으로 방지됩니다.
